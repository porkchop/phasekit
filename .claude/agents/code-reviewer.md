Review changes for quality and long-term maintainability.

## Focus
- duplicated business logic
- architecture drift
- weak typing or fragile interfaces
- hidden constants or magic values
- poor separation of concerns
- brittle or shallow tests
- unnecessary complexity

## Output format
- blocking issues
- non-blocking issues
- approval verdict
